# CatCare 校园猫咪社区 - 技术文档

## 1. 项目概述

CatCare 是一个校园猫咪社区管理系统，采用前后端分离架构，帮助校园里的爱猫人士记录和管理猫咪信息及投喂记录。

### 1.1 技术栈

| 层次       | 技术选型                    |
| -------- | ----------------------- |
| 前端框架     | Vue 3 (Composition API) |
| 构建工具     | Vite                    |
| 路由管理     | Vue Router              |
| HTTP 客户端 | Axios                   |
| 后端框架     | Flask                   |
| 数据库 ORM  | Flask-SQLAlchemy        |
| 数据库迁移    | Flask-Migrate           |
| 跨域处理     | Flask-CORS              |
| 数据库      | MySQL                   |

## 2. 数据库设计

### 2.1 数据库模型

#### Cat 模型（猫咪档案）

```python
class Cat(db.Model):
    __tablename__ = 'cats'
    id = db.Column(db.Integer, primary_key=True)
    name = db.Column(db.String(100), nullable=False)
    description = db.Column(db.Text, nullable=True)
    status = db.Column(db.String(50), nullable=False, default='在校')
    image_url = db.Column(db.String(255), nullable=True)
```

| 字段          | 类型          | 必填 | 说明               |
| ----------- | ----------- | -- | ---------------- |
| id          | Integer     | 是  | 主键，自增            |
| name        | String(100) | 是  | 猫咪名字             |
| description | Text        | 否  | 猫咪描述             |
| status      | String(50)  | 是  | 状态（在校/毕业），默认"在校" |
| image\_url  | String(255) | 否  | 图片路径             |

#### FeedingRecord 模型（投喂记录）

```python
class FeedingRecord(db.Model):
    __tablename__ = 'feeding_records'
    id = db.Column(db.Integer, primary_key=True)
    cat_id = db.Column(db.Integer, db.ForeignKey('cats.id'), nullable=False)
    food_type = db.Column(db.String(100), nullable=False)
    time = db.Column(db.DateTime, nullable=False)
```

| 字段         | 类型          | 必填 | 说明            |
| ---------- | ----------- | -- | ------------- |
| id         | Integer     | 是  | 主键，自增         |
| cat\_id    | Integer     | 是  | 外键，关联 cats.id |
| food\_type | String(100) | 是  | 食物类型          |
| time       | DateTime    | 是  | 投喂时间          |

### 2.2 关系说明

- `feeding_records.cat_id` 外键关联 `cats.id`
- 删除猫咪时，会级联删除其所有投喂记录
- 删除猫咪时，会同时删除服务器上存储的图片文件

## 3. 后端 API 接口

### 3.1 基础配置

- 后端服务地址：`http://127.0.0.1:5000`
- 跨域：启用 CORS，允许前端跨域请求

### 3.2 API 列表

| 方法     | 路径                           | 说明             |
| ------ | ---------------------------- | -------------- |
| GET    | `/api/test`                  | 测试连接           |
| GET    | `/api/cats`                  | 获取所有猫咪         |
| POST   | `/api/cats`                  | 添加新猫咪（支持文件上传）  |
| PUT    | `/api/cats/<id>`             | 更新猫咪信息         |
| DELETE | `/api/cats/<id>`             | 删除猫咪（含图片和投喂记录） |
| GET    | `/api/feeding`               | 获取最近100条投喂记录   |
| POST   | `/api/feeding`               | 添加投喂记录         |
| GET    | `/api/cats-with-records`     | 获取今日有喂养记录的猫咪   |
| GET    | `/static/uploads/<filename>` | 访问上传的图片        |

### 3.3 接口详情

#### 1. GET /api/cats - 获取所有猫咪

**响应示例：**

```json
[
  {
    "id": 1,
    "name": "小白",
    "description": "白色的小猫咪",
    "status": "在校",
    "image_url": "/static/uploads/xxx.jpg"
  }
]
```

#### 2. POST /api/cats - 添加新猫咪

**请求格式：** `multipart/form-data`

| 参数          | 类型     | 必填 | 说明        |
| ----------- | ------ | -- | --------- |
| name        | String | 是  | 猫咪名字      |
| description | String | 否  | 猫咪描述      |
| status      | String | 否  | 状态，默认"在校" |
| file        | File   | 否  | 图片文件      |

**响应示例：**

```json
{
  "id": 2,
  "name": "花花",
  "description": "三花猫咪",
  "status": "在校",
  "image_url": "/static/uploads/xxx.jpg"
}
```

#### 3. PUT /api/cats/<id> - 更新猫咪信息

**请求格式：** `multipart/form-data`

| 参数          | 类型     | 必填 | 说明    |
| ----------- | ------ | -- | ----- |
| name        | String | 否  | 猫咪名字  |
| description | String | 否  | 猫咪描述  |
| status      | String | 否  | 状态    |
| file        | File   | 否  | 新图片文件 |

#### 4. DELETE /api/cats/<id> - 删除猫咪

**响应示例：**

```json
{
  "message": "删除成功"
}
```

**说明：** 同时删除关联的投喂记录和服务器上的图片文件。

#### 5. GET /api/feeding - 获取投喂记录

**说明：** 返回最近100条记录，按时间倒序排列。

**响应示例：**

```json
[
  {
    "id": 1,
    "cat_id": 1,
    "food_type": "猫粮",
    "time": "2026-05-13T12:00:00"
  }
]
```

#### 6. POST /api/feeding - 添加投喂记录

**请求格式：** `application/json`

| 参数         | 类型      | 必填 | 说明                  |
| ---------- | ------- | -- | ------------------- |
| cat\_id    | Integer | 是  | 猫咪ID                |
| food\_type | String  | 是  | 食物类型                |
| time       | String  | 否  | 投喂时间（ISO 格式），默认当前时间 |

#### 7. GET /api/cats-with-records - 今日动态

**说明：** 返回今日有喂养记录的猫咪及其最近3条喂养记录。

**响应示例：**

```json
[
  {
    "id": 1,
    "name": "小白",
    "image_url": "/static/uploads/xxx.jpg",
    "feedings": [
      {
        "id": 1,
        "food_type": "猫粮",
        "time": "2026-05-13T12:00:00"
      }
    ]
  }
]
```

### 3.4 图片上传

- 上传目录：`backend/static/uploads/`
- 文件命名：使用 UUID 生成唯一文件名
- 访问方式：`http://127.0.0.1:5000/static/uploads/<filename>`

## 4. 前端页面

### 4.1 路由配置

| 路径         | 组件          | 说明   |
| ---------- | ----------- | ---- |
| `/`        | Home.vue    | 首页   |
| `/cats`    | Cats.vue    | 猫咪档案 |
| `/feeding` | Feeding.vue | 投喂记录 |

### 4.2 页面功能

#### Home.vue - 首页

**功能：**

- 显示项目标题 "CatCare"
- 两个导航卡片按钮：猫咪档案、投喂记录
- 最新动态区域：展示今日有喂养记录的猫咪卡片

**交互：**

- 导航卡片默认黑底白字，悬停时反色
- 猫咪卡片悬停时上移 4px + 阴影位移
- 图片固定高度，卡片高度一致

#### Cats.vue - 猫咪档案

**功能：**

- 显示所有猫咪列表
- 添加新猫咪（名字、描述、状态、图片上传）
- 行内编辑猫咪信息
- 删除猫咪（带确认弹窗）

**交互：**

- 所有输入框、按钮高度统一 40px
- 文件上传使用自定义按钮，隐藏原生控件
- 表单元素圆角为 0，无边框高亮
- 点击编辑时切换为输入框模式
- 操作成功后局部刷新数据

#### Feeding.vue - 投喂记录

**功能：**

- 选择猫咪
- 输入食物类型
- 选择投喂时间（可选）
- 显示最近100条投喂记录

## 5. UI 设计规范

### 5.1 色彩方案

| 用途   | 颜色值                 |
| ---- | ------------------- |
| 背景   | `#ffffff`           |
| 文字   | `#000000`           |
| 边框   | `1px solid #000000` |
| 次要文字 | `#666666`           |
| 辅助文字 | `#999999`           |
| 分隔线  | `#eeeeee`           |

### 5.2 布局规范

- 页面容器居中，最大宽度：Home 页面 800px，其他页面 700px
- 所有元素圆角：`border-radius: 0`
- 所有表单元素高度：`40px`
- 无边框高亮：`outline: none`
- 保持大量留白，增加呼吸感

### 5.3 交互样式

- 导航卡片：默认黑底白字，悬停反色，过渡 0.3s
- 猫咪卡片：悬停上移 4px，右下方 4px 黑色阴影
- 操作按钮：极简文本链接样式，下划线

### 5.4 自定义光标

- 全局使用猫爪图标作为光标
- 点击激活状态使用另一套光标样式
- 滚动条、表单元素均应用自定义光标

## 8. 开发规范

### 8.1 代码组织

- 模型定义在 `models.py`
- 路由和业务逻辑在 `app.py`
- 配置在 `config.py`
- 前端页面在 `src/views/`

### 8.2 避免循环导入

**模型初始化顺序（正确）：**

```python
# models.py
db = SQLAlchemy()  # 空实例，延迟初始化

# app.py
app = Flask(__name__)
app.config.from_object(Config)
from models import db, Cat, FeedingRecord  # 先导入模型
db.init_app(app)                           # 再初始化
migrate = Migrate(app, db)                 # 最后创建 Migrate
```

**说明：** 确保模型在 `Migrate` 创建前被导入，以便 Flask-Migrate 能检测到 schema 变化。
